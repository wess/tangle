Architecture

Two processes, one Postgres, bare repos on disk.

  • ✦API process — bun src/server.ts on :3000 handles JSON, the git Smart-HTTP wire protocol, and attachment uploads.
  • ✦Web process — bun src/web/serve.ts on :3001 serves the SPA and proxies /api/* to the API.
  • ✦Caddy out front — routes *.git/* straight to the API, everything else through the web container, with automatic Let's Encrypt.
  • ✦Bare git repos on disk — standard format under REPO_DIR, recoverable with plain git.
  • ✦Postgres + pluggable storage — the system of record, with blobs in S3 or local. No telemetry, no CLA.
Caddy (:80 · :443) │ ┌─────────┴──────────┐ *.git/* everything else │ │ API (:3000) web (:3001) ──► /api/* │ ┌────┴─────┬──────────────┐ bare git Postgres S3 / local (REPO_DIR) (records) (blobs)
Get started

Running in a few minutes.

Docker Compose is the recommended path — clone, set a few env vars, and bring it up. The first signup becomes the owner.

terminal 
# 1. clone
git clone https://github.com/wess/tangle && cd tangle

# 2. configure
cp .env.example .env
# set DOMAIN (or :80 for HTTP), POSTGRES_PASSWORD, SECRET, RESEND_API_KEY

# 3. run
docker compose up -d
open http://localhost   # first signup becomes the owner

Prefer to run it locally with Bun? Two processes, hot-reloaded.

local development 
# install dependencies
bun install

# API on :3000, web on :3001
bun run dev

# or run a single process
bun run api
bun run web
bun run mcp   # MCP server for AI assistants
